墨痕技能

进化记录运维

Evolution Ledger · 七轮沉淀

功能摘要

README 进化记录管理范式。以「7 条有效记录→吸收覆盖→7×7=49 次进化→自然退出」为核心的进化沉淀方法论。每一次修改都成为进化的一部分，直到自然退出。

七轮沉淀法则

7 条有效进化记录为一轮。满 7 条后压缩吸收为一条「七轮摘要」。经过 7 轮（49 条）进化后自然退出。

目录类型匹配

根据 README 目录类型（技术/写作/运维）自动匹配内容重点。技术类侧重架构，写作类侧重进展，运维类侧重变化。

使用记录强制

每次使用 README 必须记录操作内容、触发者、变更范围、影响评估。不记录等于没做。

实现方式

纯 Agent 范式。配套 `templates/目录README_template.md` 模板 + `references/README管理运维.md` 参考文档。Agent 自动判断目录类型并生成对应格式的进化记录。

为什么做这个

文档最怕什么？放在那里发霉。 进化记录运维的哲学是：每一份 README 都是活文档。它应该像软件一样有版本、有更新日志、有生命周期。 「7 条吸收为 1 条，7×7=49 自然退出」——这个设计来自诩墨的直觉。7 是一个神奇的数字，太多记不住，太少不够用。7 轮之后，文档已经充分进化，可以归档了。 这个范式最牛逼的地方在于：它把「写文档」从一次性劳动变成了持续进化的过程。不是写完就完了，而是每次使用都留下痕迹。
