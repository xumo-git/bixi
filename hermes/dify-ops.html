Dify 运维

Dify Ops · 知识库排障

功能摘要

Dify 知识库的全程运维技能。容器排障（API 502 / 重启循环）、nginx 修复（DNS resolver 配置）、密码重置、ssrf_proxy 修复。通过 SSH 远程操作 Docker 容器，确保知识中台 7×24 稳定运行。

容器排障

API 502 错误快速诊断：nginx 配置 → DNS resolver → 后端容器健康检查 → Docker 日志分析。三步定位问题源。

nginx 修复

Docker 环境下 nginx 的 DNS resolver 配置是常见坑点。`resolver 127.0.0.11 valid=30s` 一行配置解决容器间通信超时。

密码重置

Dify admin 密码忘记？直接操作 PostgreSQL 数据库重置。`docker exec -it dify-db psql ...` 一行命令搞定。

SSRF 防护

配置 ssrf_proxy 防止 Server-Side Request Forgery 攻击。知识库 API 调用的安全网关。

实现方式

纯 Agent 技能（SKILL.md）。通过 SSH 连接 xumo@192.168.31.11 执行 Docker 命令。标准运维流程封装为可复用的 Agent 指令。

为什么做这个

Dify 是墨家军的知识中台——所有 Agent 的知识检索都走这里。它挂了，整个知识体系就瘫了。 Dify 运维技能是在一次次故障中总结出来的。最经典的是那次 502 事件：Dify 的 Web 服务突然全部返回 502，查了半天发现是 nginx 的 DNS resolver 配置问题——Docker 容器用 hostname 互访需要配置 resolver，否则 nginx 解析不了其他容器的 hostname。 一行配置解决的问题，查了两小时。自那以后，我把所有踩过的坑都写进了 Dify 运维技能里，确保下次出问题能在 5 分钟内定位。 知识中台的稳定 = AI 系统的大脑健康。
