盘丝技能

部署技能

Site Deploy · 一键上线

功能摘要

盘丝洞网站的全自动部署系统。交互菜单（本地同步 / GitHub 推送 / 全流程）+ 同步范围（对比仅新章节 / 全量替换）。Markdown→HTML 自动转换，包含阅读进度条、TTS 语音朗读、导航链接等读者体验优化。

交互菜单

运行脚本后依次询问：操作模式（本地同步 / GitHub 推送 / 全流程）+ 同步范围（仅更新新章节 / 全量重新生成）。

Markdown→HTML

源文件是 .md，网站是 .html——自动格式转换。阅读进度条 / TTS 语音 / 章节导航全部自动生成。

增量同步

对比源目录和网站目录差异，仅同步新章节或变更内容。避免全量部署带来的性能浪费。

GitHub Pages 集成

同步完成后自动 push 到 GitHub，利用 GitHub Pages 发布。一键完成。

实现方式

Python 脚本 `scripts/sync_pansi.py` + 纯技能。源目录 `/Volumes/Mac1T/盘丝洞/` → 网站目录 `/Volumes/Mac1T/墨码/GitHub管理/pansi/`。Markdown 转 HTML 引擎 + Git 操作封装。

为什么做这个

盘丝洞网站是个静态站，用 GitHub Pages 托管。每次更新章节需要：改 Markdown → 转 HTML → 推送到 GitHub。手动做这些重复劳动很烦。 部署技能就是把这个流程一键化了：运行脚本→选模式→选范围→搞定。 最得意的设计是「交互菜单」——不是简单的参数化脚本，而是有对话感的交互式菜单。每次运行都问你要做什么、要做到什么程度，像有个部署向导在陪你操作。 增量同步也是个关键细节：只更新变动的章节，不用每次都全量重新生成。三本书加一起上百章了，全量转换要 10 秒，增量只要 2 秒。
