模型用量监控

Token Tracker · 费用洞察

功能摘要

AI 模型用量与费用的实时追踪系统。直接从 Hermes state.db（SQLite）读取数据，按模型、日期、来源（cron / feishu / terminal）三个维度聚合分析。每一分钱花在哪，一目了然。

多维聚合

按模型聚合：DeepSeek / GPT-4o / Qwen3-VL 各花了多少。按日期聚合：今天 / 本周 / 本月趋势。按来源聚合：cron 自动任务 vs 飞书对话 vs 终端调试。

费用换算

根据各模型 token 单价自动计算费用。输入 token + 输出 token 分开计算，精确到分。

终端报告

格式化的终端输出报告，支持 1 天 / 7 天 / 30 天范围。随时 `model-usage` 查看用量快照。

实现方式

Python 脚本 `model_usage.py`。读取 Hermes state.db（SQLite）中的 `usage` 表，SQL 聚合查询，格式化为终端友好输出。定时任务可自动推送用量报告到飞书。

为什么做这个

AI 用多了最怕什么？月底一看账单傻眼。 模型用量监控最开始只是个小脚本：查一下 Hermes 的 SQLite 数据库，看看今天用了多少 token。后来发现，光看 token 不够，还要看费用——不同模型价格差 10 倍，一个 Qwen3-VL 调用等于 100 次 DeepSeek。 再后来加了来源维度：cron 自动任务花了多少？飞书聊天花了多少？终端调试花了多少？——这样就能发现，哦原来每天光自动写小说就跑了几十万 token。 这就是模型用量监控：让每一分 AI 算力花得明明白白。
